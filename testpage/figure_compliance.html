WHATWG 4.4.12 Figure Element Compliance Test

The <figure> element represents self-contained content, optionally with a caption.

Test 1: Basic figure with image and caption

Figure with image content and figcaption at bottom.

Expected: Image displayed, caption below, figure indented from margins.

Test 2: Figure with caption first

Figcaption can appear before the content (per spec).

Expected: Caption above, then image.

Test 3: Figure without caption

Figure can have no figcaption (just flow content).

Expected: Just the image, indented like a figure.

Test 4: Figure with code listing (spec example)

Using figure for code listings with caption.

Expected: Caption, then code block with preformatted text.

Test 5: Figure with blockquote

Using figure for quoted content with attribution.

Expected: Quote displayed, then citation caption.

Test 6: Figure with poem (spec example)

Using figure for poetry with citation.

Expected: Poem text with line breaks, then citation.

Test 7: Multiple images in one figure

Figure can contain multiple images with shared caption.

Expected: Three images, one caption for all.

Test 8: Nested figures (spec example)

Figures can be nested for group captions.

Expected: Outer caption, then individual figures with own captions.

Test 9: Figure with styled figcaption

Figcaption can contain inline formatting.

Expected: Caption with bold, italic, and link.

Test 10: Figure with ID for referencing

Figures can be referenced by ID (see spec example).

Expected: Figure with ID that could be linked to.
